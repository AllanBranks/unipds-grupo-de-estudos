# Contribuindo com o Laboratório

Este laboratório possui caráter colaborativo e Open Source.

Nosso objetivo é aprender Engenharia de Software aplicada à Inteligência Artificial através da construção de soluções reais.

Como a participação é voluntária e rotativa, buscamos organizar o projeto de forma que qualquer colaborador consiga compreender o contexto e contribuir em qualquer momento.

---

# Princípios

## Aprender construindo

Todo projeto desenvolvido pelo laboratório possui um objetivo técnico e um objetivo de aprendizagem.

Mais importante do que entregar software é compreender as decisões tomadas durante sua construção.

---

## Conhecimento compartilhado

O conhecimento não deve permanecer apenas com quem implementou uma funcionalidade.

Cada contribuição deve deixar conhecimento suficiente para que outros colaboradores consigam compreender, evoluir ou reutilizar aquela implementação.

---

## Documentação como contexto

A documentação faz parte da arquitetura do projeto.

Ela deve fornecer contexto suficiente para que um desenvolvedor ou um agente de IA consiga compreender como determinada funcionalidade foi construída.

Sempre que possível, documente o problema, a solução adotada e as decisões tomadas.

---

## Desenvolvimento Guiado por IA

Este laboratório incentiva o uso de Inteligência Artificial durante todas as etapas do desenvolvimento.

A IA deve ser utilizada como ferramenta de apoio para:

- Pesquisa
- Arquitetura
- Desenvolvimento
- Revisão de código
- Testes
- Documentação

---

# Organização do Trabalho

O projeto é organizado através do GitHub Projects.

Cada atividade nasce como uma Issue e representa uma oportunidade de contribuição.

As atividades devem possuir contexto suficiente para permitir que qualquer colaborador possa desenvolvê-las de forma independente.

---

# Fluxo de Desenvolvimento

Cada contribuição deverá seguir, preferencialmente, o fluxo abaixo.

```text
GitHub Project

↓

Issue

↓

Implementação

↓

Pull Request

↓

Code Review

↓

Apresentação

↓

Merge
```

---

# Pull Requests

Todo Pull Request deve conter:

- Descrição do problema resolvido
- Resumo da implementação
- Documentação complementar quando necessário
- Informações suficientes para que outro colaborador consiga compreender a solução

Sempre que possível, incluir também o plano utilizado durante o desenvolvimento, permitindo que outros colaboradores ou agentes de IA consigam reproduzir o processo.

---

# Compartilhamento de Conhecimento

Após a conclusão de uma atividade, o responsável deverá apresentar ao grupo:

- O problema resolvido
- A solução adotada
- As decisões técnicas
- Os desafios encontrados
- Os aprendizados obtidos

O objetivo é compartilhar conhecimento e manter todos os participantes alinhados com a evolução do projeto.

---

# Evolução Contínua

O laboratório não trabalha com sprints fixas.

O projeto evolui continuamente conforme o interesse, disponibilidade e contribuição dos participantes.

As prioridades poderão ser redefinidas durante os encontros do grupo.

---

# Em Construção

Este documento representa a forma atual de trabalho do laboratório.

As diretrizes poderão evoluir conforme novos aprendizados forem surgindo durante o desenvolvimento das iniciativas.
